### Verification and Testing Method

To verify the tool's effectiveness in preventing search engine indexing, we use a **Searchability Comparison Test**. This empirical method proves that while a page is indexed, specific hidden parts remain invisible.

#### The Experiment Setup

1.  **Preparation**: On a publicly crawlable page, we place two distinct markers.
2.  **Marker A (The Control)**: A unique plaintext string that Google will naturally index.
3.  **Marker B (The Hidden Target)**: A unique ciphertext string contained within a URL generated by HideText.
4.  **Wait**: Allow 1-2 weeks for search engine bots (Googlebot) to crawl and index the site.

#### Verification Steps

*   **Search for Marker A**: Searching for `Plaintext1_fwfim13873` should return this blog post. This proves the page **is** indexed.
*   **Search for Marker B**: Searching for the ciphertext `FjREPxkjKhksNgYUfmVsR3E5RjwvOA==` should return **zero results**. This proves the specific content is **not** indexed despite being on an indexed page.

#### Why it Works
This success is the result of three layers of protection:
*   **XOR Obfuscation**: Breaks the semantic relationship of the text.
*   **X-Robots-Tag: noindex**: Explicitly tells bots not to store the landed URL.
*   **Canonical Tags**: Consolidates ranking to the main tool, ignoring parameterized variations.
